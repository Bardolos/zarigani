Zarigani

Welcome to the introductory Zarigani page. You will find a brief description as well as an example of one of Zarigani's many uses.

What is Zarigani?

Literally speaking, Zarigani is a species of crayfish endemic to Japan. While this aplication is colored like one, and features a suggestive icon, it has nothing to do with seafood.

Zarigani (the application) is written in Ruby and its main purpose is to transform Markdown into Markup (namely, HTML) in real time, side by side.

This isn't really impressive, but Zarigani allows for more than that: you can write the <head> wrapper in a separate file, in which you're free to include styling and functionality - like CSS and Javascript, much like you would on a real web page. Zarigani works as a surrogate browser, allowing you to incorporate content from the web (like images, javascript files, etc.) into your Markdown, with the immediate result being visibl ein HTML.

The motivation is quick prototyping. Write the <head> - called a decorator in Zarigani - once with all the style and functionality, then write all the Markdown you need, and that wrapper will be automatically envelop it. Zarigani supports changing both the decorator's contents, as well as the specific decorator file dynamically - meaning you can use Zarigani and style your HTML with your favourite editor at the same time as you go. Simply press Refresh or start typing to apply the changes.

Zarigani is currently in advanced beta.

Zarigani in action

To see Zarigani in action, it would be better to try it for yourself. Unfortunately I cannot make the code available yet for personal reasons.

Feel free to look at the following screenshot, and take of it what you will:

You might notice the style is quite different from this one - it's just using a different decorator. The one in this picture was written by me specifically for one purpose: to document things. More specifically, to document video game development things. Even more specifically, to document video game quests in development.

For this, I put together a semi-formal CSS, with a clear structure and more or less even coloring - without too many moving parts. For added flexibility, I added a few Javascript/JQuery code. This code turns nested markdown tags into interactive dialog sequences - useful for quests that branch within the plot, or that are heavy on exposition. With some creativity, you can make dialoge, quotes or journal entry stand out by using clever styling on current Markdown tokens.

What is Zarigani's future?

As stated, Zarigani is currently in an advanced beta state: it works, but there's improving to do.

Regarding the quest documentation system, a user interface might be in order. To be able to link quest chapters dynamically would also be a useful feature.

All these may be due, but it is difficult to know, as Zarigani currently does what it was initially made for.
